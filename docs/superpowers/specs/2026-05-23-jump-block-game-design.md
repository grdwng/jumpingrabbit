# 跳跳小方块 — 游戏设计规格文档

**日期**: 2026-05-23
**版本**: v1.0
**状态**: 已批准

---

## 1. 概念与愿景

一款轻松可爱的3D跳跳小游戏。玩家控制一个卡通小人在悬浮方块间跳跃，收集金币、能量块和爱心。游戏追求「易上手、有深度、视觉治愈」的体验——每次跳跃只有一格，但方块布局变化丰富，加上三种奖励的策略组合，让简单操作产生多层乐趣。

---

## 2. 设计语言

### 美术风格: Cartoon 卡通风格

- **整体感觉**: 柔和、圆润、轻松愉快，像一个彩色糖果世界
- **参考感**: 纪念碑谷 + 治愈系休闲游戏的结合

### 色彩系统

| 用途 | 颜色 | 说明 |
|------|------|------|
| 主色调 | `#7EC8E3` (天蓝) | 天空背景、主要UI |
| 方块色 | `#FFE5A0` (暖黄) | 地基方块 |
| 小人色 | `#FF7B9C` (珊瑚粉) | 玩家角色 |
| 金币色 | `#FFD700` (金) | 金币奖励 |
| 能量色 | `#B39DDB` (淡紫) | 能量块 |
| 爱心色 | `#FF6B8A` (玫红) | 爱心/生命值 |
| 草地/地面 | `#98D9A4` (薄荷绿) | 地面装饰 |
| 强调色 | `#FF4D6D` (草莓红) | 危险/失败提示 |
| 文字 | `#2D3047` (深灰蓝) | 主文字 |

### 字体
- 标题: **Fredoka One** (圆润可爱，适合卡通风格)
- 正文: **Nunito** (圆润易读)
- 备选: system-ui, sans-serif

### 3D 视觉规格
- 相机: 跟随视角，第三人称背后视角
- 视角高度: 俯视约30度角，能看到前方方块布局
- 人物大小: 约0.8格（不遮挡前方视线）
- 方块规格: 1.2x0.5x1.2 (宽x高x深)，边缘圆润处理
- 阴影: 软阴影，方块下方淡淡阴影增加立体感

### 可自定义选项

**背景主题**:
| 主题 | 描述 |
|------|------|
| 蓝天白云 | 默认，天蓝渐变，棉花糖云朵 |
| 夕阳余晖 | 橙红渐变，热带风格 |
| 星空夜景 | 深蓝+星星，安静神秘 |
| 森林绿意 | 绿色主调，清新自然 |
| 糖果色彩 | 粉紫渐变，梦幻感 |

**方块配色**: 每个关卡可使用不同主色，由关卡数据定义

---

### 动画与动效
- **跳跃动画**: 弹道曲线，ease-out 效果，约 400ms
- **落地动画**: 轻微弹跳 (squash & stretch)
- **收集动画**: 物品飞向UI，伴随粒子效果
- **失败动画**: 小人掉落，淡出，然后重置
- **UI 过渡**: 300ms ease-in-out

## 4. 核心功能

### 4.1 跳跃机制

**操作**: 方向键 (↑↓←→)
**跳跃距离**: 固定 1 格
**跳跃方向**: 上/下/左/右 四个方向
**跳跃时间**: 400ms (可调整)

**跳跃规则**:
1. 按下方向键 → 小人向该方向跳跃
2. 跳跃动画播放期间，输入无效（防止连跳）
3. 落在方块上 → 收集奖励，重置等待输入
4. 跳空 (周围1格内没有方块) → 触发失败

### 4.2 方块系统

**方块类型**:

| 类型 | 外观 | 说明 |
|------|------|------|
| 地基方块 | 暖黄色，实心 | 正常站立，可收集奖励 |
| 起点方块 | 绿色边框 | 每关起点位置 |
| 终点方块 | 闪烁边框 + 旗子 | 到达后过关 |

**方块奖励** (随机生成):

| 奖励 | 颜色 | 效果 |
|------|------|------|
| 金币 | 金色圆形 | +100分 |
| 能量块 | 紫色菱形 | 累计5个触发护盾(防一次跳空) |
| 爱心 | 红色桃心 | +1生命(上限5) |

### 4.3 生命值系统

- 初始生命: 3颗心
- 跳空: -1心，失败动画，回到关卡起点
- 收集爱心: +1心 (上限5)
- 生命归零: 游戏结束，显示结算画面，可重新开始本关或返回关卡选择

### 4.4 关卡进度

- 总关卡: 20关 (MVP范围)
- 每关目标:
  - 到达终点方块
  - 收集足够的金币 (通过门槛)
- 通关条件: 到达终点 + 金币达标
- 通关评价: 根据生命值剩余和金币总数给1-3星

### 4.5 能量块技能

- 累计5个能量块 → 自动激活护盾
- 护盾效果: 下一次跳空不死亡，自动消耗护盾
- 护盾显示: 小人周围发光轮廓

### 4.6 通关庆祝系统

每关通关时触发完整的庆祝流程：

**庆祝动画序列** (约3秒):
1. 画面轻微放大 + 闪光效果 (0.3秒)
2. 角色跳舞动画 + 星星粒子爆炸 (1秒)
3. 分数/星星飞入结算框 (0.5秒)
4. 展示评价和奖励内容 (1.2秒)

**通关奖励**:
- 金币结算: 收集的金币 × 通关评价倍数 (1星=1x, 2星=1.5x, 3星=2x)
- 额外奖励: 每关首通奖励 50金币，连击奖励 (连续通关奖励)
- 星星解锁: 新关卡解锁，永久进度保存

**彩蛋系统**:
- 每5关有一个特殊彩蛋关 (5, 10, 15, 20)
- 彩蛋关通关触发特殊动画 + 大量金币雨 + 专属角色动作
- 特定条件触发隐藏彩蛋 (如：零死亡通关、收集所有爱心等)
- 彩蛋内容随机：小烟花、彩虹、金币雨、角色特殊动作

## 6. 技术架构

### 渲染引擎
- **Three.js**: 3D渲染，WebGL
- 单HTML文件，包含内联JS/CSS，便于分享

### 核心模块

```
Game
├── Renderer (Three.js 场景管理)
├── Camera (跟随相机系统)
├── InputManager (方向键处理)
├── LevelManager (关卡数据，加载)
├── PlayerController (小人控制，跳跃)
├── BlockGenerator (方块生成，奖励分配)
├── ScoreManager (分数，生命，能量积累)
├── UIManager (HUD，结算，菜单)
├── AudioManager (音效，背景音乐)
└── ParticleSystem (收集特效，跳跃粒子)
```

### 关键实现

**方块布局生成**:
- 每关预设方块坐标数组
- 奖励随机分配 (金币70%，能量20%，爱心10%)
- 终点固定在关卡末端

**跟随相机**:
- 相机位置 = 小人位置 + 固定偏移 (0, 5, 8)
- lerp 平滑跟随，factor 0.1
- 只在看小人不阻挡视野的前提下跟随

**碰撞检测**:
- 跳跃结束时检测小人脚下1格内是否有方块
- 有方块: 正常落地
- 无方块: 判定跳空

## 7. 音效与音乐

### 音效
- 跳跃: 轻快的「咚」声
- 收集金币: 清脆的「叮」
- 收集能量: 科技感的「嗡嗡」
- 收集爱心: 温馨的「噗」
- 落地: 轻微的「啪」
- 过关: 欢快的胜利旋律
- 死亡: 下降音效 + 重置音

### 背景音乐
- 循环轻松的卡通风格音乐
- 音量适中，不干扰游戏
- 静音选项

## 8. 关卡设计

### 关卡难度梯度

| 阶段 | 关卡 | 特点 |
|------|------|------|
| 入门 | 1-3 | 直线距离，方块密集，无陷阱 |
| 基础 | 4-6 | 引入转向，稀疏方块 |
| 进阶 | 7-10 | 需要跳过空隙，节奏变快 |
| 挑战 | 11-15 | 陷阱方块(跳跃后消失)，多路径选择 |
| 高难 | 16-20 | 极限跳转，组合陷阱 |

### 每关数据结构

```javascript
{
  id: 1,
  name: "起步",
  blocks: [
    {x: 0, z: 0, type: "start"},
    {x: 1, z: 0, type: "normal", reward: "coin"},
    {x: 1, z: 1, type: "normal", reward: "energy"},
    ...
  ],
  endPosition: {x: 5, z: 3},
  targetCoins: 300
}
```
