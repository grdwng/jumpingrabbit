# 护盾系统与得分系统修复设计

> 日期: 2026-05-25

## 1. 概述

修复跳跳小方块游戏中的护盾系统和得分系统问题，确保：
- 护盾在 HUD 中正确显示
- 护盾消耗后能正确重新开始本关卡
- 得分系统正确区分本关得分和累积得分

## 2. 问题分析

### 2.1 护盾显示问题
- `onFall()` 中使用 `baseY` 变量但未定义，导致位置重置失败

### 2.2 护盾消耗逻辑缺陷
- 消耗护盾后只设置 `gameState = 'waiting'`，未重新加载关卡
- player 位置未重置到起始点

### 2.3 得分系统混乱
- 当前 `score` 既表示关卡得分又表示累积得分
- 通关时加分逻辑不清晰

## 3. 修复方案

### 3.1 数据结构

```javascript
// 添加关卡得分字段
this.levelScore = 0;      // 本关卡得分
this.totalScore = 0;      // 累积总分

// 护盾消耗后重新开始
this.loadLevel(this.currentLevel);
```

### 3.2 护盾消耗流程

```
消耗护盾 → 播放保护音效 → 重新加载当前关卡 → 保留 shields/coins/energy
```

### 3.3 得分计算

| 事件 | 得分 |
|------|------|
| 收集金币 | 不加分（金币单独计数） |
| 通关本关 | +100 分到 `levelScore` |
| `totalScore` | = 所有关卡 `levelScore` 之和 + 金币 × 20 |

### 3.4 HUD 显示

```
❤️×3   💎×3/5   ⚡️×1   |   关卡 1   |   💰×5   ⭐×150
生命   能量      护盾           金币      累积总分
```

## 4. 修改文件

- `index.html`: 游戏逻辑修改

## 5. 验收标准

- [ ] 护盾 ⚡️×0 在 HUD 正确显示
- [ ] 消耗护盾后 player 重置到起始位置，本关卡重新开始
- [ ] 通关显示本关得分 100 分
- [ ] 累积总分 = 所有关卡得分之和 + 金币 × 20
